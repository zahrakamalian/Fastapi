<h3> ۱. API چیه؟</h3>
API مخفف <b>Application Programming Interface</b> هست.  
به زبان ساده: یه جور منو یا در پشتی بین دو برنامه است.
مثلاً وقتی تو اپلیکیشن اسنپ‌فود غذا سفارش می‌دی، اپلیکیشن تو با API رستوران حرف می‌زنه و می‌گه: «این غذا رو می‌خوام، آدرس اینه». رستوران هم جواب می‌ده: «اوکی، ۳۰ دقیقه دیگه می‌رسه».
بدون API، برنامه‌ها نمی‌تونن با هم حرف بزنن و اطلاعات رد و بدل کنن. API پل ارتباطی بین برنامه‌هاست.
</div>
<div dir="rtl">
<h3> ۲. FastAPI چیه؟</h3>
FastAPI یک کتابخونه/فریم‌ورک پایتون است که کمک می‌کنه خیلی راحت و سریع API بسازی.

ویژگی‌های مهمش به زبان ساده:
- خیلی سریع است (تقریباً مثل زبان‌های سریع‌تر از پایتون)
- کدت رو می‌نویسی → خودش مستندات قشنگ و تعاملی درست می‌کنه (نیازی به نوشتن داکیومنت جدا نیست)
- اشتباهات رو زود بهت نشون می‌ده (چون از نوع داده‌ها استفاده می‌کنه)
- راحت می‌تونی باهاش کار همزمان (async) انجام بدی
- یادگیریش نسبتاً ساده است (اگر پایتون بلد باشی)

مثل اینه که بخوای یک مغازه آنلاین بزنی؛ FastAPI بهت کمک می‌کنه خیلی سریع مغازه‌ت رو بسازی، قفسه‌ها رو مرتب کنی، مشتری‌ها راحت سفارش بدن و حتی خودش به صورت خودکار کاتالوگ و راهنما (مستندات) برات درست کنه.
</div>

<h3> ۳. Endpoint چیه؟</h3>
Endpoint یعنی <b>آدرس مشخص</b> یک کار خاص در API.
هر endpoint معمولاً:
<ul>
<li>یک مسیر (path) دارد → مثل `/users` یا `/products/iphone`
</li>
<li>یک متد HTTP دارد → GET (گرفتن اطلاعات)، POST (ارسال اطلاعات)، PUT (ویرایش)، DELETE (حذف)</li>
</ul>

<h3> ۴. Swagger چیه؟</h3>
Swagger (یا دقیق‌تر: <b>Swagger UI</b> یک صفحه وب تعاملیه که FastAPI به صورت خودکار برات درست می‌کنه.

وقتی سرور FastAPI رو اجرا می‌کنی و می‌ری به آدرس:
http://127.0.0.1:8000/docs
یه صفحه باز می‌شه که:
- همه endpointها رو لیست کرده
- می‌تونی پارامتر بدی و دکمه Execute بزنی
- جواب رو همونجا می‌بینی
- حتی می‌تونی مستندات رو بخونی

</div>

<h3> ۵. دو نوع اصلی ارسال پارامتر: Query Parameter و Path Parameter</h3>

| نوع               | مثال در URL                              | مثال در کد FastAPI                              | توضیح ساده                                      | کی استفاده کنیم؟                              |
|-------------------|------------------------------------------|--------------------------------------------------|--------------------------------------------------|-------------------------------------------------|
| **Path Parameter** (پارامتر مسیر) | `/users/42` <br> `/products/iphone-16`  | `@app.get("/users/{user_id}")`                 | پارامتر **داخل آدرس** قرار می‌گیره            | وقتی بخشی از **شناسه اصلی** است (مثل آیدی، نام کاربر، کد محصول) |
| **Query Parameter** (پارامتر کوئری) | `/users?age=25&city=tehran` <br> `/search?q=کفش&sort=price` | `@app.get("/users")` <br> `def get_users(age: int = None, city: str = None)` | پارامتر **بعد از علامت ?** و با & جدا می‌شن   | برای **فیلتر**، **جستجو**، **صفحه‌بندی**، **تنظیمات اختیاری** |

مثال واقعی:
<ul>
<li>/users/123               → حتماً باید آیدی کاربر رو بدی (Path)</li>
<li>/users?role=admin&limit=10 → می‌تونی فقط ادمین‌ها رو با ۱۰ تا محدودیت ببینی (Query)</li>
</ul>

هر دو رو FastAPI خیلی راحت و با چک کردن نوع داده‌ها مدیریت می‌کنه.
</div>
